# PRD 文档

本目录存放产品需求类文档。

PRD 的核心目的不是堆叠描述，而是让需求更清楚，让代码更可验证，让错误更早暴露，让重构更安全。

## 适合放入

1. 产品需求说明。
2. 用户故事与使用场景。
3. 功能范围与不做事项。
4. MVP 边界与阶段目标。

## 不适合放入

1. 详细技术实现方案。
2. 测试报告或验收记录。
3. 发布流程或 Git/PR 操作说明。
4. 临时需求讨论、执行 todo 或一次性过程记录。

## 命名建议

使用 `序号-主题.md`，例如 `001-agent协作规范需求.md`。

## 推荐结构

PRD 建议按以下结构扩展：

1. `背景`：说明产品问题、用户价值、当前风险和为什么需要该需求。
2. `产品目标`：使用 SMART 和 BDD 场景描述目标，关键目标可写成 Gherkin。
3. `非目标`：明确首版不做什么。
4. `核心用户故事`：按用户类型、使用场景和验收标准展开。
5. `用户体验原则`：说明交互、反馈、状态可见性和边界提示。
6. `数据与权限边界`：说明默认收集、默认不收集、存储、删除和权限规则。
7. `状态模型`：列出核心状态、含义和流转约束。
8. `技术方案`：只写影响产品边界和验收的技术方案，不替代 design 文档。
9. `首版验收门禁`：列出必须满足的可验证验收项。
10. `待确认问题`：列出需要产品、研发、QA 或运营继续确认的问题。
